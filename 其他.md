
# 常见思维模式

## ReAct 模式（真 Agent）

```
Thought: xxx
Action: xxx
Observation: xxx
→ 重复
```

特点：
- 固定思考格式
- 强依赖 LLM 按格式输出
- 只是“思考+行动”的那一段逻辑

示例：
- Thought: 我需要先创建 main.py，因为它是入口
- Action: write_file main.py ...
- Observation: 文件已创建

- Thought: 接下来要安装依赖
- Action: bash npm install
- Observation: 安装成功

## Agentic Loop 对照
步骤	你的 Agent 模式	现在任务管道	状态
1	读取需求	Tab1 上传文档 / 填写描述	✅ 有
2	读取项目结构	无，AI 看不到代码库	❌ 缺
3	思考怎么做	意图识别 + 开发计划	🔶 手动触发，非自主
4	生成/编辑文件	Tab4 发消息，但 AI 无法真正写文件	❌ 缺工具
5	运行命令测试	无	❌ 缺
6	看报错	无	❌ 缺
7	修复代码	无	❌ 缺
8	循环直到完成	无自动循环，全靠人工推进	❌ 缺

Plan Mode 是纯应用层实现，和 SDK 没有关系。

本质上只有两种机制：

1. Prompt 控制（最常用）

"此阶段只输出计划，不要调用任何文件修改工具"
AI 遵从指令，只读不写。

2. 工具列表控制（更严格） 发消息时不传入 write/edit 类工具，AI 物理上无法改文件。

Claude Code 自己的 Plan Mode 也是这么做的：

系统提示里加了 "plan before you act" 类指令
提供了一个特殊工具 ExitPlanMode，AI 规划完了调这个工具通知应用层"计划已就绪"
用户在 UI 上点确认后，再开启完整工具集发第二条消息执行
这个 ExitPlanMode 工具就是 Claude Code 自己注册的，不是 Anthropic API 的原生能力。

所以如果 DevX 的 Pipeline 要做类似机制，完全是在 buildCodingPlanMessage 的 prompt 里写"只规划不执行"，然后等 AI 回复完成后展示确认按钮，用户点确认再发 buildCodingExecuteFromPlanMessage。整个逻辑在 TaskPipelinePanel.tsx 里就能闭环，不依赖任何外部 SDK 接口。
